## **1. Пример с `int` и передачей по значению**

```csharp
int b = 6;
F(b);
Console.WriteLine(b);

static void F(int a)
{
    a = 5;
}
```

### **1) Сколько объектов выделится в динамической памяти?**

**Нисколько.**
Тип `int` — это тип-значение, живёт на стеке (временной памяти).

---

### **2) Сколько ячеек будет выделено под переменные?**

**Две ячейки:**

* одна под `b`
* одна под параметр `a` при вызове `F`

Функция `F` не является переменной → под неё память для переменной не выделяется.

---

### **3) Изменит ли строка `a = 5` переменную `b`?**

**Нет.**
`a` — отдельная копия значения `b`, существующая только внутри функции.
После выхода из `F` переменная `a` уничтожается, `b` остаётся прежним.

**Результат на экране: `6`.**

---

### **4) Почему у `F` тип возвращаемого значения — `void`?**

Потому что функция ничего не возвращает, она просто выполняет действие.
Она не передаёт наружу никакое значение.

## **2. Пример с одинаковыми именами переменных**

```csharp
int a = 6;
F(a);
Console.WriteLine(a);

static void F(int a)
{
    a = 5;
}
```

### **1) Ссылаются ли два `a` на одну и ту же ячейку памяти?**

**Нет.**
Это две разные переменные:

* внешняя `a`
* внутренняя `a`, созданная для параметра функции

Они живут в разных стеках (внешний — в `Main`, внутренний — в `F`).

---

### **2) Изменится ли внешняя `a`? Что выведется?**

**Не изменится.**
Параметр — копия значения.

**Выведется: `6`.**

---

## **3. Пример со строками (`string`)**

```csharp
string a = "hello";
F(a);
Console.WriteLine(a);

static void F(string b)
{
    b = "world";
}
```

### **1) Сколько объектов создаётся в динамической памяти?**

**Два объекта:**

* объект строки `"hello"`
* объект строки `"world"`

Строки всегда живут в куче.

---

### **2) Изменится ли объект `"hello"`?**

**Нет.**

* Строки неизменяемы (`immutable`)
* Перезапись ссылки *не изменяет объект*, она только меняет указатель переменной

---

### **3) Изменится ли адрес в переменной `a`?**

**Нет.**
`b` — отдельная переменная, указывающая на копию ссылки.
Она переписывается, но `a` остаётся прежней.

**Выведется: `"hello"`.**

---

## **4. Пример со `struct`**

```csharp
A b = new()
{
    value = 1,
};
F(b);
Console.WriteLine(b.value);

static void F(A a)
{
    a.value = 2;
}

struct A
{
    public int value;
}
```

### **1) Изменится ли `value` внутри `b`?**

**Нет.**

`struct` — это тип-значение.
Передача параметра — копирование *всего содержимого структуры*.
В функции `F` переменная `a` независима от `b`.

Вывод будет: `1`.

---

### **2) Изменится ли что-либо, если назвать переменную `b` именем `a`?**

**Нет.**
Имена не влияют на то, что это две разные переменные.

---

### **3) Что будет, если заменить `struct` на `class`?**

Если `A` станет **классом**:

* переменные будут хранить **ссылку** на объект
* `new` создаст **один объект в куче**
* и `b`, и параметр `a` будут ссылаться на этот же объект

Поэтому:

* `a.value = 2` изменит объект в куче
* вывод будет **2**

**То есть передаётся копия ссылки, но объект один и тот же.**
